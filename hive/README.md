# Hive (Metastore)

> В процессе развертывания я столкнулся с одной нетривиальной проблемой: Spark (Worker и Master) работали штатно, до того пока не было принято решение подключить Hive Metastore, чтобы облегчить интегрирование с другими сервисами пайплайна. Проблема была в том что после подключения Hive Metastore перестал работать целый кодовый модуль интеграции с S3-протоколом для общения с MinIO S3.    
> Модуль S3: `org.apache.hadoop:hadoop-aws:3.3.4` (также доп. модуль для AWS: `com.amazonaws:aws-java-sdk-bundle:1.12.262`)  
> Причина была проста, при вызове Spark'ом класса `S3AFileSystem` из модуля S3, Hive обращаясь внутри своего контейнера к данному классу попросту не находил его, так как он не был вшит в Hive Metastore контейнер.

> После "подтягивания" версии в контейнер Hive, данной ошибки не появлялось, но возникла другая - более тривиальная: Запрос класса `IOStaticTransport` у Hadoop модулей. Из-за несовместимости версий пакета: `hadoop-aws-3.3.0` и версии экосистемы Hadoop внутри контейнера (`3.1.0`) возникал конфликт, к моему счастью, последний.
